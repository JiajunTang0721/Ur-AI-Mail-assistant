# Mail assistant

Mail assistant 是一个面向 Gmail 的智能邮箱整理助手。它以 Chrome Extension + FastAPI 后端的方式运行，负责从 Gmail 读取邮件、结合用户偏好和历史反馈做结构化分析，并将结果映射到前端的邮件看板中展示。

系统的核心目标不是简单“分类邮件”，而是把以下两个维度明确拆开：

- 板块归属：邮件应该展示在哪个板块
- 优先级：邮件本身的重要程度有多高

## 核心能力

- Gmail 邮件同步与基础字段规范化
- LLM 特征抽取
- 本地优先级模型打分
- 四板块展示：
  - 优先内容
  - 48h
  - 待处理
  - 可忽略
- 已完成邮件流转：
  - 24h 内已完成
- 用户反馈闭环：
  - 更重要
  - 少显示
- 单封邮件重新概述，不影响板块和优先级
- 用户偏好设置：
  - 重点寄件人
  - 关注方向顺序
  - 自定义关注内容

## 系统设计原则

### 1. 板块归属和优先级是两个独立维度

系统不会把“高优先级 = 优先内容”这种关系写死。

例如：
- 一封邮件可以位于 `48h`，但优先级是 `medium`
- 一封邮件可以位于 `待处理`，但优先级是 `high`
- `ignore` 级邮件只会进入 `可忽略`

### 2. 优先级来自本地模型

系统会将邮件转化为结构化特征向量，并通过本地模型输出：

- `priority_score`：0 ~ 1
- `priority_level`：`high | medium | low | ignore`

### 3. 板块归属来自独立逻辑

系统单独决定邮件应当进入哪个板块：

- `priority_content`
- `within_48h`
- `todo`
- `ignore`

## 前端展示逻辑

当前前端会将分析结果以卡片形式展示，并支持：

- 导航切换不同板块
- 卡片颜色体现优先级
- 查看原邮件
- 重新概述
- 已完成 / 恢复为未完成
- 更重要 / 少显示

部分相似邮件会被聚合显示，用户可展开查看每一封具体邮件。
