# 问题与修复记录

这份文档记录的是项目推进过程中已经出现过、并且已经修过的代表性问题。它的目的不是追责，而是帮助后续开发时少走回头路。

## 1. 扩展注入与上下文问题

### 1.1 `Receiving end does not exist`

**现象**

- popup 或扩展图标点击后，提示 content script 没有接收到消息

**原因**

- 当前 Gmail 标签页尚未注入 content script
- 老页面未刷新，扩展刚重载但 Gmail 页面仍在跑旧脚本

**解决**

- `background.ts` 增加内容脚本补注入与重试
- 明确要求每次重载扩展后强制刷新 Gmail 页面

### 1.2 `Extension context invalidated`

**现象**

- Gmail 页面报错
- 面板挤压、按钮失效、旧面板和新面板叠在一起

**原因**

- 扩展重载后，页面里残留旧 content script 构建出来的 DOM

**解决**

- 新注入前清理旧 `lma-ext-root`
- 建议在重载扩展后整页刷新 Gmail
- 对错误做显式提示，而不是静默失败

## 2. OAuth 与配置问题

### 2.1 OAuth Client ID 配了但仍无法授权

**现象**

- 面板一直显示“先连接 Gmail”
- 或提示缺少 `VITE_GOOGLE_OAUTH_CLIENT_ID`

**原因**

- `.env` 中 `client_id` 填错
- 后缀被重复写成 `.apps.googleusercontent.com.apps.googleusercontent.com`
- Google Cloud 的 `Item ID` 与当前扩展 ID 不一致
- 当前账号不在 `Test users`

**解决**

- 增加了更明确的错误说明
- 构建侧对重复后缀做清理
- 文档明确区分 `Chrome Extension` client 和 `Web application`

### 2.2 PowerShell 里 `pip` 与 `uvicorn` 不能直接运行

**现象**

- `pip.exe 无法运行：拒绝访问`
- `uvicorn` 无法识别

**原因**

- 当前 Windows 环境没有正确把可执行文件暴露到 PATH

**解决**

- 固定使用 `C:\ProgramData\anaconda3\python.exe -m pip`
- 固定使用 `C:\ProgramData\anaconda3\python.exe -m uvicorn`

## 3. 后端联调问题

### 3.1 端口占用导致后端起不来

**现象**

- `WinError 10048`
- 启动失败但不容易意识到旧进程仍在

**原因**

- 旧的 FastAPI 进程仍占着 `8010`

**解决**

- 提供端口排查与结束进程的命令
- 后续联调统一以 `8010` 为默认端口

### 3.2 后端 500

**现象**

- 扩展显示“后端分析接口返回 500”

**原因**

- 邮件内容中的异常字符触发序列化问题
- OpenAI `json_schema` 格式不符合要求

**解决**

- 对输入做清洗
- 修正 `response_format.json_schema`
- 保留规则引擎兜底

### 3.3 `422`：`analysisMode` 不兼容

**现象**

- 新增“重新概述”后，旧后端只接受 `incremental / rerank`

**原因**

- 前后端版本阶段性不一致

**解决**

- 扩展侧新增兼容逻辑
- 单封邮件重新概述在旧后端也可通过兼容模式完成

## 4. Gmail 与 LLM 结果问题

### 4.1 同一批旧邮件被反复送入 LLM

**现象**

- 刷新或重新打开助手后，同样邮件再次被分析
- 同一封邮件在不同时间出现不同摘要

**原因**

- 缺少账号级历史缓存与邮件列表签名判断

**解决**

- 引入 `chrome.storage.local` 历史缓存
- 按账号保存 `analysisCache + analysisHistory`
- 只有新邮件进 LLM，旧邮件优先复用历史

### 4.2 用户偏好对排序影响不明显

**现象**

- 勾选 `招聘` 或 `学业` 后，优先级变化太弱

**原因**

- 规则加权不够强
- LLM 提示词里对偏好的结构化约束不足

**解决**

- 在规则层提高关注方向与发件人命中权重
- 在 LLM 提示中明确把偏好与历史作为排序参考
- 保持多选项等权

### 4.3 原邮件跳转不准确

**现象**

- `查看原邮件` 像演示按钮
- 点击后不能可靠跳到目标 Gmail 邮件

**原因**

- 早期只做了页面内模拟点击

**解决**

- 优先读取 Gmail 列表真实链接
- 不可用时回退到 thread 地址

## 5. 交互与视图问题

### 5.1 反馈或重排后直接跳回顶部

**现象**

- 用户点击反馈或重新分析后，面板突然跳回顶部

**原因**

- 重新渲染后没有保留滚动视口

**解决**

- 引入视口快照
- 在重排后执行恢复和跟随逻辑

### 5.2 卡片移动后用户看不到它滑到哪里

**现象**

- 卡片位置变了，但用户丢失目标

**原因**

- 只有重排动画，没有视口跟随

**解决**

- 增加卡片跟随锚点
- 当目标卡重排时，视口平滑移动到新位置

### 5.3 渐变遮罩挡住内容或导致无法滚动

**现象**

- 顶部渐变层影响滚动和点击

**原因**

- 遮罩层默认始终显示且拦截事件

**解决**

- 仅在滚动后显示渐变
- 取消遮罩层对滚动区事件的干扰

### 5.4 按钮玻璃效果过粗糙

**现象**

- hover 效果更像简单高亮，而不是材质变化

**原因**

- 缺少共享高光层、边缘反射、柔和 blur 和胶囊滑移

**解决**

- 重构按钮 hover 视觉层
- 引入共享玻璃指示器和更细的材质层次

## 6. 当前仍需注意的边界

虽然上面的问题已经修复，但还有两个重要边界需要记住：

### 6.1 后端一旦关闭，扩展会退化

- 本地 FastAPI 是真实 LLM 分析的前提
- 后端关闭时，扩展可做本地重排兜底，但无法完成新的 LLM 摘要

### 6.2 历史缓存目前仍主要是本地账号态

- 目前主要保存在扩展本地
- 不是服务端统一账户历史
- 换电脑或换浏览器 profile 时不能天然同步
