# 功能拆解清单

这份文档把仓库里的功能按用户路径和代码层拆开，方便 GitHub 读者快速理解项目到底能做什么。

## 1. Gmail 接入相关

- 打开 Gmail 后加载右侧助手
- 通过扩展弹窗触发 Gmail 授权
- 使用 `gmail.readonly` 读取邮箱数据
- 拉取最近邮件列表和线程信息

## 2. 邮件分析相关

- 规范化 Gmail 邮件字段
- 结合本地偏好、历史、反馈一起分析
- 输出中文展示字段：
  - `displaySender`
  - `briefSubject`
  - `aiSummary`
  - `aiReason`
- 输出双维度结果：
  - `board_type`
  - `priority_score / priority_level`
- 生成解释标签和排序分数

## 3. 面板交互相关

- 面板展开/收起
- 四个主板块切换
- 邮件详情联动
- 发件人筛选
- 关注方向筛选
- 自定义关注项输入
- 统计概览展示

## 4. 反馈与学习相关

- 标记“更重要”
- 标记“少显示”
- 保存单封邮件反馈计数
- 重新分析整批邮件
- 刷新单封邮件摘要
- 将历史反馈用于下一轮优先级偏置

## 5. 本地存储相关

- `chrome.storage.local` 保存账号级扩展状态
- `backend/data/preferences.json` 保存默认偏好
- `backend/data/feedback.json` 保存反馈模板或本地反馈数据

## 6. 后端能力相关

- 健康检查
- 分析邮件批次
- 重新分析
- 单封邮件重概述
- 读写偏好
- 记录反馈
- 拉取账号反馈记录

## 7. 目前最适合对外展示的功能亮点

- Gmail 侧边助手不是静态稿，而是扩展注入
- AI 输出不是自由文本，而是结构化特征
- 板块归属和优先级被明确拆成两个维度
- 反馈可以参与后续优先级判断
- 仓库已经包含联调文档和展示素材

## 8. 当前仍在迁移中的部分

- 独立原型页仍保留旧 tab 名称
- 部分版本化文档更偏目标态设计
- 展示仓库已经整理，但后续仍可继续统一命名与注释
